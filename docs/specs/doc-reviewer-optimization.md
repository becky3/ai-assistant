# doc-reviewerサブエージェント最適化

## 概要

doc-reviewerサブエージェントのレビュー時間・粒度を最適化し、fullレビューとdiffレビューを使い分けられるようにする。

## 背景

### 現状の問題点

PR #142 での実測データ:
- 所要時間: 129秒（2分超）
- API往復回数: 8回
- トークン消費: 35,445
- レビュー対象: 2ファイル（軽微な修正）
- 有用な指摘: 1件、対応不要: 多数

### 問題の原因

1. **常にフルレビューが実行される**: 軽微な修正でも全観点のチェックが走る
2. **プロジェクト方針の未考慮**: プロジェクト全体で統一している事項も個別に指摘される
3. **スキップ基準の不明確さ**: CLAUDE.mdで必須手順とされているため、軽微な修正でもスキップしづらい

## ユーザーストーリー

- 開発者として、軽微なドキュメント修正時には変更箇所のみの高速レビューを受けたい
- 開発者として、新規仕様書作成時には網羅的なフルレビューを受けたい
- 開発者として、レビューモードを明示的に指定して使い分けたい

## 技術仕様

### レビューモードの定義

| モード | 用途 | レビュー範囲 | 目標時間 |
|--------|------|-------------|---------|
| `full` | 新規作成・大幅改訂 | ファイル全体を全観点でチェック | 60秒以下 |
| `diff` | 軽微な修正・レビュー指摘対応 | 変更差分のみを関連観点でチェック | 30秒以下 |

### モード選択の判断基準

#### `full` モードを使用するケース
- 新規仕様書の作成時
- 既存仕様書の大幅改訂時（構造変更、セクション追加など）
- 明示的に `full` を指定された場合

#### `diff` モードを使用するケース
- 誤字脱字の修正
- 既存内容の微修正・補足追加
- PRレビュー指摘への対応
- 明示的に `diff` を指定された場合

### 呼び出し方法

```
# fullレビュー（デフォルト・従来動作）
doc-reviewerサブエージェントで docs/specs/f1-chat.md をレビューしてください

# fullレビュー（明示指定）
doc-reviewerサブエージェントで docs/specs/f1-chat.md をfullレビューしてください

# 差分レビュー
doc-reviewerサブエージェントで docs/specs/f1-chat.md を差分レビューしてください
doc-reviewerサブエージェントで docs/specs/f1-chat.md をdiffレビューしてください
```

### 差分レビューの処理フロー

1. **変更差分の取得**
   - `git diff HEAD~1 -- <file>` で直近の変更差分を取得
   - ステージング済みの場合は `git diff --cached -- <file>` も考慮

2. **変更タイプの判定**
   - 追加行のみ: 新規内容のレビュー
   - 削除行のみ: 削除の妥当性確認
   - 変更行: 変更前後の比較レビュー

3. **観点の絞り込み**
   - 構造変更がない場合: 構造の完全性チェックをスキップ
   - AC変更がない場合: AC品質チェックをスキップ
   - 関連ファイル変更がない場合: 整合性チェックをスキップ

4. **レポート生成**
   - 変更箇所に対する指摘のみ出力
   - 既存部分への指摘は含めない

### 許容パターンの設定

プロジェクト方針として許容される項目を定義し、指摘対象外とする:

| 項目 | 許容理由 |
|------|---------|
| エージェント定義の `model` フィールド省略 | 全エージェントで統一的に省略（デフォルトモデル使用） |
| サブエージェント仕様書での簡略化された技術仕様 | エージェント定義ファイルとの役割分担 |
| レトロスペクティブでのPR番号不在 | 開発中PRの場合は番号未確定 |

### CLAUDE.mdのスキップ基準

ドキュメントレビューのスキップ基準を明確化:

| 変更内容 | レビュー要否 | 推奨モード |
|---------|-------------|-----------|
| 誤字脱字のみの修正 | スキップ可 | - |
| PRレビュー指摘への対応 | 推奨 | diff |
| 既存内容の軽微な補足 | 推奨 | diff |
| 新規セクションの追加 | 必須 | full |
| 新規仕様書の作成 | 必須 | full |

## 実装変更

### 1. エージェント定義の更新

`.claude/agents/doc-reviewer.md` に以下を追加:

```markdown
## レビューモード

レビュー依頼時に「差分レビュー」「diffレビュー」というキーワードが含まれる場合は差分レビューモードで実行する。
それ以外はフルレビューモードで実行する。

### 差分レビューモード

1. `git diff` コマンドで変更差分を取得
2. 変更箇所に関連する観点のみをチェック
3. 変更箇所に対する指摘のみを出力

### フルレビューモード

従来通り、ファイル全体を全観点でチェックする。
```

### 2. 仕様書の更新

`docs/specs/doc-review-agent.md` の「処理フロー」セクションに差分レビューモードを追加。

### 3. CLAUDE.mdの更新

「Claudeによる実装完了時の必須手順」の「ドキュメントレビュー」項目にスキップ基準を追記。

## 受け入れ条件

- [ ] AC1: 「差分レビュー」「diffレビュー」キーワードで差分レビューモードが起動する
- [ ] AC2: 差分レビューモードでは変更箇所のみがレビュー対象となる
- [ ] AC3: 差分レビューモードの実行時間が30秒以下である
- [ ] AC4: フルレビューモードは従来通り動作する
- [ ] AC5: モード未指定時はフルレビューがデフォルトで実行される
- [ ] AC6: 許容パターンに該当する項目は指摘されない
- [ ] AC7: CLAUDE.mdにスキップ基準が明記されている

## テスト方針

### 差分レビューモードのテスト

- [ ] 誤字修正のみの変更で差分レビューを実行し、30秒以下で完了するか
- [ ] 変更箇所以外への指摘が出ないか
- [ ] git diffが正しく取得されるか

### フルレビューモードのテスト

- [ ] フルレビュー指定で従来通りの動作をするか
- [ ] モード未指定でフルレビューが実行されるか

### 許容パターンのテスト

- [ ] modelフィールド省略が指摘されないか
- [ ] その他の許容パターンが指摘されないか
